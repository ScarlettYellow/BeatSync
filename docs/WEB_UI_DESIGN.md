# BeatSync Web服务 UI设计方案

## 一、设计原则

- **极简主义**：界面简洁，功能明确
- **易用性**：操作流程清晰，用户无需学习
- **响应式**：适配不同屏幕尺寸

### 2.2 详细设计说明

#### 页面标题
- **位置**：页面顶部居中
- **文字**："BeatSync 视频对齐工具"
- **样式**：大号字体（24-28px），加粗，深色

#### 上传区域1：原始视频
- **标题**："原始视频 (Dance Video)"
- **说明文字**："选择文件" 或 "拖拽文件到此处"
- **按钮**：文件选择按钮（标准样式）
- **已选择文件显示**：
  - 显示文件名和大小
  - 格式：`已选择: {文件名} ({文件大小})`
  - 可选：显示文件图标或预览

#### 上传区域2：音源视频
- **标题**："音源视频 (BGM Video)"
- **其他同上**

#### 开始处理按钮
- **位置**：两个上传区域下方，居中
- **状态**：
  - **未就绪**：灰色，禁用（两个文件都未上传）
  - **就绪**：蓝色/绿色，可点击（两个文件都已上传）
  - **处理中**：灰色，禁用，显示"处理中..."
- **样式**：大按钮（宽度80%，高度50px），圆角

#### 处理状态显示
- **位置**：开始处理按钮下方
- **内容**：
  - 等待中：`处理状态: 等待中...`
  - 处理中：`处理状态: 正在处理，请稍候...`
  - 完成：`处理状态: 处理完成！`
  - 失败：`处理状态: 处理失败`

#### 下载结果按钮
- **位置**：处理状态下方
- **状态**：
  - **处理完成前**：隐藏或灰色禁用
  - **处理完成后**：显示，可点击
- **样式**：与开始处理按钮相同

## 三、交互流程设计

### 3.1 正常流程

1. **页面加载**
   - 显示两个上传区域
   - "开始处理"按钮禁用（灰色）
   - "下载结果"按钮隐藏

2. **上传原始视频**
   - 用户点击"选择文件"或拖拽文件
   - 显示文件名和大小
   - 检查是否两个文件都已上传

3. **上传音源视频**
   - 同上

4. **两个文件都上传后**
   - "开始处理"按钮变为可点击（蓝色/绿色）
   - 显示提示："两个文件已就绪，可以开始处理"

5. **点击"开始处理"**
   - 按钮变为"处理中..."（灰色，禁用）
   - 处理状态显示："正在处理，请稍候..."
   - 页面显示加载动画（可选）

6. **处理完成**
   - 处理状态显示："处理完成！"
   - "下载结果"按钮显示并可用
   - "开始处理"按钮恢复（可选：重置为初始状态）

7. **点击"下载结果"**
   - 自动下载处理后的视频文件
   - 文件名：`beatsync_result_{时间戳}.mp4`

### 3.2 错误处理

1. **文件格式错误**
   - 上传时即提示："不支持的文件格式，请上传MP4或MOV格式"
   - 不显示文件名

2. **处理失败**
   - 处理状态显示："处理失败"
   - "开始处理"按钮恢复可点击
   - 允许用户重新上传或重新处理

3. **网络错误**
   - 显示："网络错误，请重试"
   - 按钮恢复可点击

## 四、视觉设计建议

### 4.1 配色方案

- **主色调**：蓝色系（#007AFF 或 #0066CC）
- **成功色**：绿色（#28A745）
- **错误色**：红色（#DC3545）
- **背景色**：浅灰色（#F5F5F5）或白色
- **文字色**：深灰色（#333333）

### 4.2 字体

- **标题**：24-28px，加粗
- **正文**：16px，常规
- **按钮文字**：18px，加粗

### 4.3 间距

- **页面边距**：20-30px
- **元素间距**：30-40px
- **上传区域内部间距**：20px

### 4.4 圆角

- **按钮**：8-12px
- **上传区域**：8px
- **整体容器**：12px（可选）

## 五、响应式设计

### 5.1 桌面端（>768px）
- 两列布局或单列居中
- 上传区域宽度：80%，最大600px
- 按钮宽度：80%，最大400px

### 5.2 移动端（≤768px）
- 单列布局
- 上传区域宽度：95%
- 按钮宽度：95%
- 字体适当缩小

## 六、可选增强功能（后续）

1. **文件预览**：上传后显示视频第一帧
2. **进度条**：处理时显示进度（如果改为异步）
3. **处理历史**：显示最近处理记录
4. **拖拽上传**：支持拖拽文件到上传区域

## 七、技术实现建议

### 7.1 前端技术
- **HTML5**：基础结构
- **CSS3**：样式和动画
- **JavaScript（原生）**：交互逻辑
- **Fetch API**：与后端通信

### 7.2 关键功能实现

1. **文件上传**
   - 使用 `<input type="file">`
   - 支持拖拽（`drag and drop` API）
   - 文件验证（格式、大小）

2. **状态管理**
   - 使用简单的状态变量
   - 根据状态更新UI

3. **API调用**
   - 上传文件：`POST /api/upload`
   - 处理视频：`POST /api/process`
   - 下载结果：`GET /api/download`
